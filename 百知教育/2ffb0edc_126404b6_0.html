第1讲 单行函数

1.  基本概念及预备常识

a)        所谓单行函数就是作用与表里的每一行数据，执行一次的函数。

   eg：select employee_id,first_name,abs(salary) from employees;

    abs()是计算绝对值的函数。这条sql执行后，会返回107条结果，abs会针对employees

表里的每一行计算一次，得到一个结果。

b)       DUAL表

     一行一列的特殊表，属于SYS用户。如果我们只需要函数执行一次，可以使用

个表。dual表里的数据没有任何意义，只是为了维护select语句的完整性。

     eg : select abs(-10) from ???;

     如果问号处我们写入employees，那么该条查询语句会针对表里的每一行执行一次。虽然abs(-10)与表里数据没有任何关系，但仍然会返回107行结果。如果填入dual，那么因为dual表里只有一行数据，所以该条查询语句只会有一个结果。

2.  字符串相关函数

l  length(str) --获取字符串str的长度

  eg：查询员工的姓、及姓的长度

      select last_name,length(last_name) from employees;

      查姓的长度是5的员工

      select * from employees where length(last_name) = 5;

l  字符串连接

    select 'a' || 'b' from dual;

    也可以使用函数 select concat('a','b') from dual;

l  substr(str,begin,length)

对str做截取，从begin(下标从1开始)开始截length位

    eg：select substr( 'abcdefg' , 2 , 4 ) from dual;

    对’abcdefg’取子串，从第二个元素开始取4位，得到的结果为 ：

  

       -- 查找姓名是以'S'开头的员工信息

     select employee_id,fist_name,salary from employees where substr(first_name,1,1)='S';

l  instr(str1,str2,begin) -- 在str1里查找str2，从begin（下标从1开始）开始查找。找到返回str2首字母所在的下标，否则返回-1。

eg： 查找姓中含有'g'的员工信息

select employee_id,fist_name,salary from employees where instr(first_name,'g',1)!=-1;

l  lower(str) , upper(str) -- 将str转换成小写/大写

eg：查找姓是以'k'或者'K'开头的员工信息

select employee_id,fist_name,salary from employees

where lower( substr(first_name,1,1) ) = 'k' ;

---> 先计算substr，其结果作为lower函数的参数代入运算，直至得到最终结果

注意：与java一样，函数可以嵌套调用，只要参数类型匹配即可。执行时，从最内层

      开始，由里向外，逐层计算。

3.  数学相关函数

l  mod(num1,num2) -- 取模，num1%num2

    eg：打印员工编号为基数的员工信息

        select * from employees where mod(employee_id,2)=1;

l  trunc(n,length) --对数字n进行截断，精度到小数点后length位

l  round(n1, length) --对数字n四舍五入，精度到小数点后length位

eg：select round(123.45678,3) from dual --123.457

select round(123.45678,0) from dual --123

--查询工号、姓、工资、日薪

       select employee_id,last_name,salary,round(salary/22,2) from employees

l  abs(num) --对num取绝对值

l  获取随机数：dbms_random.random() , 产生一个很大的随机数（可正，可负）

eg：请随机产生一个正整数

     select dbms_random.random() from dual;

     请随机产生一个100以内的正整数

     select mod( abs(dbms_random.random()) , 100) from dual;

     请随机查询employees表里的一行数据

     select employee_id,first_name,salary from employees

     where employee_id= mod(abs(dbms_random()),107)+100;

     注意：表里一共107行数据，employee_id从100开始。

4.  日期相关操作和函数

       时间与日期是数据库中常用的数据类型，在处理很多运算的时候都需要用到。比如

    银行应用在处理开户业务时要存储开户日期，电信应用在结算通讯费用是也要用到通话

    时间。下面我们就来学习oracle数据库中关于日期和实践的相关运算和函数。

l  SYSDATE表示当前系统时间

eg：1）Select sysdate from dual;  --->获取当前系统时间

    在sysdate提供的时间数据中，不仅包含年月日的信息，还包括时分秒的信息，

    甚至可以精确到毫秒。本例中我们只看到了年月日信息是因为oracle默认的日

    期显示格式省略了时分秒等额外信息。oracle里默认的日期格式为'dd-mon-rr'。

    2）显示明天的时间

        select sysdate+1 from dual

注意：对于日期进行算术运算时，默认单位是天

l add_months(myDate, num) --对myDate做月份的算术运算

eg：--显示上个月的今天

    select add_months(sysdate, -1) from dual

l  last_day(myDate) -- 获取myDate所在月份的最后1天

eg：--获取当前月份的最后1天

   select last_day(sysdate) from dual

l months_between(myDate1,myDate2) --计算两个日期之间相隔几个月，返回值为小数

l  round(myDate,日期格式字符串) --对myDate日期，根据指定日期格式做四舍五入

l trunc(myDate, 日期格式字符串) --对myDate日期，根据指定日期格式做截断

可选日期格式字符串:  year--按年截断  month-按月截断  day-星期  省略格式字符串表示截断时分秒。

eg：select trunc(sysdate,'yyyy') from dual --年的第1天

    select trunc(sysdate,'mm') from dual --月的第1天

    select trunc(sysdate,'day') from dual --星期(周)的第1天

    select trunc(sysdate) from dual; --当天的零时零分零秒

5.  类型转换相关操作和函数

l  to_number(str) --将给定的字符串str转换成数字类型值

eg：select to_number('1234') from dual; ---> 将字符串1234转换成数字1234

注意：如果给定的str不是有纯数字组成，sql无法执行，报异常。

l  to_char(myValue,'fmt') -- 将给定的myValue值按照指定格式fmt转换成字符串类型值

1）to_char(myNumber,'格式字符串') --将数字变成字符串

   常见格式：0或者9表示占位，还可以使用各种货币符号￥$等。

   eg : 请用科学计数法显示员工的工资信息

       select employee_id,first_name,to_char(salary,'$999,999.00') from employees;

   注意：当实际数值超过给定的字符串格式时，数据无法正常显示，会出现若干***。

2）to_char(myTime,'格式字符串') --将给定的日期值转换成字符串时间表示【重点】

      因为oracle默认的日期显示格式省略了时分秒等额外信息。如果我们想在现实

   时间的时候看到时分秒的信息的话，就需要手动指定日期的显示格式了。

   常见格式字符串：

   年： yy   yyyy   rr  rrrr   year(年的英文全拼)

   月： mm   mon(英文单词缩写)   month

   日： dd   ddsp(英文全拼)  ddth  ddspth(英文全拼的序数词)

   星期： d   dy(英文单词缩写)   day

   小时： hh---am，pm   hh24

       分钟：mi    秒：ss

作用：

a)        把myTime日期类型根据指定的日期格式字符串，转换成字符串显示

eg：--按照年月日时分秒的格式，显示当前日期时间

       select to_char(sysdate,'yyyy-mm-dd hh24:mi:ss day') from dual;

b)       根据给定的日期格式字符串，获取日期的不同部分

eg：--显示当前月份

select to_char(sysdate, 'mm') from dual

--显示员工的工号、姓、入职日期、入职月份

select employee_id,last_name,hire_date,to_char(hire_date,'mm') from employees

--查询当前月份入职的员工

select employee_id,hire_date from employees

       where to_char(hire_date,'mm')=to_char(sysdate,'mm')

l  to_date(strDate, 日期格式字符串) --根据给定的日期格式字符串，把字符串转成日期

eg：--输出2008-8-8是星期几

       select to_char(to_date('08-01-08','mm-dd-yy'), 'day') from dual

--查询1998年12月3日之后入职的员工

select * from emp where hiredate >= to_date('1998-12-3','yyyy-mm-dd');

 

6.  NVL函数

1)       问题引入：

     与java 语言类似，Oracle中使用NULL代表一个值不存在。需要注意的是NULL

值参与运算总返回NULL，这给我们的运算过程带来问题。当我们计算employees中每

个员工每月实发工资（工资+奖金，奖金是工资*奖金比率）时，问题就出现了：

Select employee_id , first_name , salary , commission_pct ,

(salary + salary*commission_pct) as sum

from employees；

其运行结果是 ：

       我们发现很多员工的实发工资变成了NULL，这很显然是不对的。造成这个错误的

    原因在于有的员工的commission_pct为NULL , 在计算总工资时，由于有空值参与， 

    所以导致整个运算结果为空。

2)       问题解决：

       nvl(n1,n2) --判断n1是否是null，是null执行n2，否则执行n1。(n1==null?n2:n1)

       实际上我们希望当commission_pct为NULL时，它的值按0来处理。这时我们就

    可以使用NVL函数解决此问题（将NULL转化为默认值进行计算）。我们将刚才的SQL

    改写成如下形式 ：

select employee_id , first_name , salary , commission_pct ,

(salary + salary* nvl(commission_pct,0)) as sum

from employees;
