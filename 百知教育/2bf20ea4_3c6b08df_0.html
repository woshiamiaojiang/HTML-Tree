1.  组函数

    我们前面学过的函数有一个共同的特点，就是只能对一条数据进行运算。比如to_char()在执行一次的情况下只能转化一个日期。在SQL语句中，还有另外一组函数，他们不是针对一条数据进行计算，而是对一组数据进行统计，这类函数统一称作组函数。

l  sum(列名)：求一组数据的总和

avg(列名)：求一组数据的平均值.

max(列名)：在一组数据中找出最大的一个值.

    min(列名)：在一组数据中找出最小的一个值

例如：

    Select max( salary ) from employees . 查询最高工资

    Select min(salary) from employees . 查询最低工资.

    Select avg(salary) from employees . 查询平均工资.

    Select sum(salary) from employees 查询所有员工的月薪总和.

当然，我们也可以把前四条SQL合并在一起：

    select max(salary) , min(salary) , avg(salary) , sum(salary) from employees

我们会得到如下结果：

如果我们希望平均工资只保留两位小数的话，可以使用round函数四舍五入

  select max(salary) , min(salary) ,round( avg(salary), 2) , sum(salary) from employees

 从上面的例子中我们看出，组函数是将所有查询结果看做一个分组，并针对分组中的数据进行统计，虽然一个组中有很多条数据，但统计出来的结果只有一条。

    在刚才的例子里，我们没有加任何查询条件，所以表中的所有数据都参与了分组运算。如果我们加了查询条件的话，那么就只有符合条件的数据才会参与分组运算。比如，我们可以查询90号部门的最高，最低工资，以及平均工资，总工资

select max(salary) , min(salary) ,round( avg(salary), 2) , sum(salary) from employees where department_id=90

l  count

         eg：--统计有奖金的人数

                    --统计有奖金的人数

2.  数据分组GROUP BY【难点】

 1) 语法：select ... from ... where ... group by 分组条件

 2）细节：组函数是把所有查询结果看成一个大组，然后对这个大组进行统计。比如 select max(salary), sum(salary) from employees ,这条SQL会在所有员工中统计最高工资和工资总和。很多时候，我们需要对查询结果进行更加精细的划分，比如我们想获得每个部门的最高工资和工资总和，这时我们就需要把所有查询结果再次按照部门ID分成若干个小组，然后分别对每个小组进行统计。具体过程如下图所示：

这个时候，我们就需要使用group by 子句了。我们将上面的SQL语句改写成如下形式：

Select  department_id , max(salary) , sum(salary) from employees group by department_id .

    group by 后面跟的是分组列的列名字，他会将查询结果中分组列(department_id)取值相同的数据归为一个小组，然后针对每个小组分别进行max(salary)和sum(salary)的运算. 其运行结果如下：

我们将前两幅图合并在一起，来理解一下分组运算的过

 3）举例说明：

    --统计各岗位的最高工资

              --步骤1：确认分组字段 (把岗位相同的员工，放到一个组中)

                 group by job_id

              --步骤2：写完整SQL（针对每组数据，求平均工资）

                          select max(salary) from employees group by job_id

       --统计各部门各岗位的平均工资

              --步骤1：确认分组字段 (部门相同的放到一组，部门相同再把岗位相同放到1组)

                     group by department_id,job_id

              --步骤2：写完整SQL（针对每组数据，应用组函数-求平均工资）

              select department_id,job_id,avg(salary) from employees group by department_id,job_id

4）需要强调，由于SQL语句中使用类group by 子句之后，同一小组内的多条数据最终要被合并为一条数据，那么对select 后面的列名字是有要求的。我们先来看一个错误的例子：Select department_id,first_name,max(salary),sum(salary) from employeesgroup by department_id 这条sql产生错误的原因就在于在使用group by 根据department_id 进行分组之后，又要查询first_name列，我们知道一个小组最后会被合并成一条数据，那么小组内每个人的first_name 由不相同，就会造成无法合并的问题。那么究竟哪些列可以出现分组SQL中呢，我们给出如下规则，请大家牢记：

ü  规则1：只有出现在group by中的列才能出现在select 子句中或order by子句中

ü  规则2：如果在group by子句中的列上应用了某些函数，select子句中或order by子句的相应列也必须应用同样的函数

ü  规则3：那些没有出现在group by 中的列，只有配合分组函数可以出现在select子句中 或 order by子句中

 

3.  HAVING子句：对分组后的结果进行过滤

       1）语法： select ... from ... where ... group by... having 过滤条件 order by ...

          分组操作是在where 语句之后进行的，如果我们相对分组之后的数据再次依照

   某个条件进行筛选怎么办呢？比如我们想查询平均工资大于3000的部门的部门  

   ID,最高工资，和总工资，我们该如何编写SQL呢? 首先我们来看一个错误的例子：

       Select department_id , max(salary) , sum(salary) From employees

       Where avg(salary) > 3000  Group by department_id ;

 

          这条SQL的错误在于，在where条件中使用了avg()函数。我们前面强调过，分组工作是在where判断之后进行的。也就是说在where条件执行的时候，还没有分组，既然没有分组，也就无法计算每个组的平均工资了，这条SQL是无法被Oracle 执行的。

为了解决这个问题，我们引入Having子句，having子句和where类似，也是用于指定查询条件的，但是他有一个非常显著的特点：即having子句是在分组完成之后执行的。所以上面错误的SQL应当改成：

    select department_id , max( salary) , sum(salary), from employees

    group by   department_id  having avg( salary ) > 3000;

       2） where 和 having 区别：

Ø  where : 对分组前的数据进行过滤

Ø  having: 对分组后的结果进行过滤

               如果where和having都可以实现过滤，优先使用where，因为效率高。 如果

      使用分组后的结果过滤，必须使用 having。例如：

               --统计60、70、110 各部门的平均工资

                    select department_id,avg(salary) from employees

                    where department_id in (60,70,110)  group by department_id

                    select department_id,avg(salary) from employees group by department_id

                    having department_id in (60,70,110)

4.  SELECT语句的语法顺序

    最后我们来总结一下SQL语句的编写顺序和执行顺序,正确理解SQL语句的执行顺序对我们编写复杂SQL语句有着非常重要的作用。

Ø  编写顺序：

     SELECT … FROM … WHERE … GROUP BY … HAVING … ORDER BY …

Ø  执行顺序：

    WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY

1)     通过where 条件，对表中的每个数据进行一次判断，符合条件被加入到查询结果中

2)     如果使用了group by 子句，会对查询结果进行分组操作。

3)     通过having子句，对分组之后的数据进行过滤

4)     执行select 后面的各个函数和运算得到最终结果

5)     通过order by 语句对最终结果进行排序.
