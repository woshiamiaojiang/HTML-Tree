会话跟踪

如上图：当浏览器向ServletB发出请求时，它的登陆操作已经完成了，但是却没有留下任何依据能够证明它已经成功登陆。以至于ServletB对他的登陆状态无法判别，这种情况叫用户状态的丢失。造成这种结果的原因是http协议的状态。

会话跟踪：就是追踪用户状态，简单的说就是指将用户操作过的重要业务步骤记录下来，以便在后续的处理中使用。

会话跟踪技术有2种：Cookie 和 HttpSession

Cookie

知识点

Cookie是服务器保存在用户浏览器上得一段字符串，由2部分构成：名称和值。

当浏览器向服务器发送请求时，会自动将这些服务器存储的字符串发送到服务器

我们可以利用这些字符串记录用户的重要状态，如是否登录，用户权限等

在服务器端的Servlet中，通过检查这些字符串来判断用户的状态

创建Cookie

对应的类：javax.servlet.http.Cookie

Cookie c = new Cookie(“name”,”zs”);

设置Cookie的有效期

c.setMaxAge( 24 * 60 * 60 ) ;//指定cookie在客户端浏览器上存储的有效期，以秒为单位。

当cookie的生命周期结束时，浏览器会自动删除该cookie

setMaxAge的特殊值：

-1 ：会话Cookie，浏览器关闭时自动删除

0 ： 表示要删除一个cookie

如果没有调用setMaxAge()方法，默认值为-1

往用户的浏览器中存储Cookie

response.addCookie(c) ;

当调用response.addCookie(c)的时候，Cookie并没有立即被送回到客户端，只有当本次请求的应答返回到客户端时，Cookie才会和应答内容一起到达浏览器

Cookie是否被正确的添加到浏览器中，可以通过浏览器的查看Cookie功能进行验证

以Chrome浏览器为例：设置-->显示高级设置-->隐私设置-->点击“内容设置”-->点击“所有Cookie和网站数据”，部分截图如下：

从客户端读取Cookie

Cookie[] cookies = request.getCookies();//返回类型为Cookie[] 数组

如果客户端没有任何cookie，则该函数返回null

服务器只能获取本应用向客户端中添加的Cookie

打印客户端的所有Cookie:

    Cookie[] cookies = request.getCookies();

    if (cookies != null){//读到

       System.out.println("读到!!!!!!!!!");

       for (Cookie c : cookies) {

           out.println(c.getName()+","+c.getValue()+"<br/>");

       }

    }

Cookie的特点

采用字符串的形式存储数据，简单灵活

存储在用户的浏览器上，减少对服务器的压力

 

对于重要数据而言，存储在浏览器上不够安全

无法存储格式复杂的数据格式，如对象，集合等
