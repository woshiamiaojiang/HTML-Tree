对于chain方式的跳转，其实比较复杂，涉及到两个action之间的状态传递：两个action都有自己独立的ActionContext，但action1的ActionContext的内容会被复制到action2的ActionContext中；它们所使用的ValueStack是同一个，并且在Root中会依次压入action1和action2； 如果有请求参数，那么会分别调用各自Action的请求参数拦截器给action属性赋值，action1的属性值如果发生了修改，不会影响到action2的属性值

常见问题

1.  redirect 配置提示错误

在struts.xml配置文件中，跳转方式使用redirect会在xml文件提示错误，但这实际是myeclipse的问题，只是显示错误，不影响实际运行

2.  什么时候转发，什么时候重定向？

这是一个从web阶段就经常问的问题，习惯上：

action到jsp之间一般都为转发（struts中的dispatcher）；而action到action之间用重定向。一次action请求可以认为是一次独立完整操作，而一次完整操作之内推荐使用转发，否则就使用重定向。另外就是要避免刷新页面导致的表单重复提交时，可以考虑使用重定向来进行跳转。

例如：“登录action”成功后到“查询所有联系人action”，他们代表了两次不同的操作请求，应当使用重定向；又比如“增删改联系人action“成功后到“查询所有联系人action”，他们也属于两次不同的操作请求，也应当使用重定向来跳转，还有：“安全退出action”跳转应当使用重定向来跳转到“主页面action”

最后就是要跟外部系统进行交互时，只能使用重定向。

第4讲 Struts 2分页

课程内容

在学员完成了第一天布置的作业：通讯录项目中“用户登录”功能，“查询所有联系人”功能的前提下，讲解“联系人分页”功能

Service和Dao的代码可以事先由教师提供，学员只需掌握Service的使用即可，课堂重点在于讲解Action的实现和JSP实现。可以参考如下步骤来进行讲解：

1.  分页Action的实现

 

 
 

public class FindAllAction {    

 

       private UserService userService = new UserServiceImpl(); 

 

       private String name;

 

       private String qq;

 

       private Integer cityId;

 

       private Integer pageNo = 1;

 

       private Integer pageSize = 10;

 

       private int total;

 

       private List<Contact> list;

 

       private Map<Integer, String> map;     

 

       public String execute() {

 

              User   user = (User) ActionContext.getContext().getSession().get("user");

 

        // 根据条件查询当前页记录

 

              list = userService.findContacts(name, qq, cityId, user.getUsername(), pageNo, pageSize);

 

        // 根据条件查询符合条件的记录总数

 

              int count = userService.findCount(name, qq, cityId, user.getUsername());

 

        // 计算总页数

 

              total = 0;

 

              if(count % pageSize == 0) {

 

                     total = count / pageSize;

 

              }   else {

 

                     total = count / pageSize + 1 ;

 

              }

 

        // 查询所有城市数据，为条件中的城市列表以及查询结果中的城市id->name做准备

 

              map = userService.findCityMap();

 

              return "success";

 

       }

 

 

 

}

 

其中计算总页数可以简化为，看学生是否容易理解选讲：

 
 

total = (count-1) / pageSize  + 1;

 

要接收的请求参数和要作为结果返回的list, map, total都作为Action的属性，以便从request作用域可以访问到。

添加Action配置和并配置视图findall.jsp，在findall.jsp中编写如下关键代码：

     <table border="1" width="80%">

 

           <thead>

 

                  <tr>

 

                         <td><input type="checkbox"  id="checkall"/></td>

 

                         <td>编号</td><td>姓名</td><td>手机</td>

 


                  <td>邮箱</td><td>qq</td><td>城市</td>

                  </tr>

尤其注意其中${map[c.cityId]}的写法，查询出来的联系人只有城市id，因此要将id显示为名称，需要一个map来方便结果转换，该map的key是城市id，值为城市名称。

注意：这里有一个问题，我的map类型为Map<Integer, String>，页面上如果直接写${map[2]}可能取不到值，这是因为有的版本EL表达式将2视为long整型，所以不行，而c.cityId返回的是Integer，因此可以。

3.  添加查询表单

在学员理解了上一步之后，将地址栏的搜索、分页条件改为表单形式的：

 
 

<form action="/struts2/homework/findall"  id="myform">

 

      姓名:<input type="text"  name="name"  value="${name}"/>

 

      qq:<input type="text"  name="qq"  value="${qq}"/>

 

      城市:<select name="cityId">

 

             <option value="-1">请选择</option>

 

             <c:forEach items="${map}" var="e">

 

                    <c:if test="${e.key == cityId }">

 

                           <option value="${e.key }"  selected>${e.value }</option>

 

                    </c:if>

 

                    <c:if test="${e.key != cityId }">

 

                           <option value="${e.key }">${e.value }</option>

 

                    </c:if>

 

             </c:forEach>

 

      </select>

 

      <br/>

 

      页号<input type="hidden"  name="pageNo"  id="pageNo"  value="${pageNo}"/>

 

      每页记录数<input type="hidden"  name="pageSize"  value="${pageSize }"/>

 

      <input type="submit"  value="搜索"/>

 

      <hr/>

 

</form>

 

注意要讲明白<input>的value属性以及<option>标签selected属性赋值的理由：并不仅仅是为了回显，而是要在每次页号发生改变时，其它搜索条件也要与上次的条件保持一致，才能达到正确的效果。要注意的还有讲解时应当将两个hidden的<input>修改为text便于说明。

4.  实现分页栏

 

 
 

<c:forEach begin="1"  end="${total}" var="i">

 

    <a href="javascript:void(0)"  onclick="submitform(${i})">${i}</a> &nbsp;

 

</c:forEach>

 

<script type="text/javascript">

 

    function submitform(pageNo) {

 

              document.getElementById("pageNo").value =   pageNo;

 

              document.getElementById("myform").submit();

 

}

 

</script>

 

根据total生成页号，然后给每个页号添加了单击事件，单击事件来修改在第3步表单中的页号<input>，并提交表单来完成分页。其中javascript:void(0)是确保阻止页号点击后的默认行为，不推荐使用#，在页面有滚动条时会有问题。

当然可以使用将搜索条件和分页条件跟在分页超链接之后作为get请求参数方式进行提交，效果类似，但实现稍微麻烦些。
