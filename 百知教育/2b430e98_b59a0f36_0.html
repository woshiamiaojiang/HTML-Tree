3.  IOC (Inversion Of Control)

回忆以前学习过的Servlet：它不需要程序员创建它的实例，调用它的service方法，它的生命周期由Tomcat来控制，因此可以说，Servlet的控制权被交出去了。交给谁了？Tomcat容器（也称为Servlet容器）由容器来负责创建它的实例，调用它的初始化，销毁以及service方法。

将某个类的控制权（创建、销毁）交由容器管理，这种工作方式就称为控制反转。

现有问题：service调用dao， action调用service, 都是直接自己new 了实现类的实例，这样导致层与层之间的耦合度太高，不利于维护

解决问题：将这些类的实例创建权交出去，交给spring容器来管理，从而降低层与层之间的耦合

Spring容器的与Servlet容器的区别：

²  Servlet容器 只能运行Servlet，JSP

²  Spring容器： 不依赖服务器，可以独立运行，普通java类（POJO）就可以在spring中运行，不需要继承特殊父类和实现特殊接口，在Spring中运行的类统一称为bean

4.  依赖注入（Dependency Injection）

依赖注入（Dependency Injection）概念是对控制反转概念的补充，早期提出的控制反转这个名字不能很好地体现Spring容器与其他容器之间的区别，因此又有人提出了依赖注入这一概念：Spring不仅来管理类实例的创建；并且还要负责维护类与类之间的依赖关系。

在建立类与类依赖关系时，不是去主动获取依赖类的实例，而是被动由Spring容器提供依赖类的实例。

第3讲 常见配置

课程内容

1.  init-method与destroy-method

格式：

<bean ... init-method="初始化方法名" destroy-method="销毁方法名">

对于销毁方法，必须在spring容器正常调用了close之后才会调用

 

2.  scope

spring中bean默认都是一个对象(单例 singleton)，但要注意它与之前单例模式的区别： 它是指同一个spring容器内单例的 （伪单例），    设计模式里的单例是整个jvm内单例

对于Struts2中的Action，如果也交给Spring管理，需要配置为多例，格式：

<bean ... scope="prototype"> 

3.  lazy-init

spring默认是饿汉式初始化：在spring容器初始化时，就会创建每个单例类的对象，并调用它们的init-method

如果要配置bean是懒汉式初始化，必须符合如下条件

²  <bean ... lazy-init="true">

²  依赖于这个bean的java类也必须是懒汉式的

第二章 Spring管理bean

第1讲 通过Spring管理已存在bean

课程内容

现在的需求是：将一个已经存在的类交给Spring管理，例如，java.util.Calendar，它的特点是一个抽象类，是通过一个静态工厂方法来创建实例的：

java.util.Calendar.getInstance()

方法一：利用工厂方法，配置如下：

<bean id="c1" class="java.util.Calendar" factory-method="getInstance"></bean>

方法二：利用Spring自定义工厂类，实现一个FactoryBean 接口

 
 

// 日历工厂类,专门用来创建日历对象

 

public class CalendarFactory implements  FactoryBean<Calendar> {

 

       // 工厂生产产品

 

       public Calendar getObject() throws Exception {

 

              Calendar   c = Calendar.getInstance();

 

              return c;

 

       }

 

       // 生产的产品类型

 

       public Class<?> getObjectType() {

 

              return Calendar.class;

 

       }

 

       // 生产的产品是单例吗

 

       public boolean isSingleton() {

 

              return false;

 

       }

 

}

 

Spring中配置如下：

<bean id="c2" class="day1.factory.CalendarFactory"></bean>

注意配置的虽然是工厂类，但通过 ApplicationContext.getBean("c2")得到的是工厂生产的产品对象。

如果希望获得工厂类的实例，可以通过ApplicationContext.getBean("&c2")

第2讲 自定义类型转换

课程内容

 

进行依赖注入时，比如对于日期对象，可以同ref注入引用，但更方便的办法是将日期看作是字符串进行值注入

步骤如下：

1. 编写转换器：

 
 

// 要继承转换器支持类

 

public class DateEditor extends PropertyEditorSupport {

 

       @Override

 

       // 方法参数：text 代表要转换的原始字符串

 

       public void setAsText(String text) throws IllegalArgumentException {

 

              try {                    

 

                     SimpleDateFormat   sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");

 

                     Date   d = sdf.parse(text);

 

                     // 保存转换结果

 

                     setValue(d);

 

              }   catch (ParseException e) {

 

                     SimpleDateFormat   sdf = new SimpleDateFormat("yyyy-MM-dd");

 

                     try {

 

                            Date   d = sdf.parse(text);

 

                            setValue(d);

 

                     }   catch (ParseException e1) {

 

                            setValue(null);

 

                     }

 

              }

 

       }

 

}

 

 

2. 配置转换器

 
 

<bean class="org.springframework.beans.factory.config.CustomEditorConfigurer">

 

       <property name="customEditors">

 

              <map>

 

                     <!-- 当遇到日期类型时，调用DateEditor -->

 

                     <entry key="java.util.Date" value="day2.DateEditor"></entry>

 

              </map>

 

       </property>

 

</bean>

 

 

3. 在某个日期属性需要注入值时，可以使用

<property name="birthday"value="1999-7-1"/>

第3讲 配合外部properties注入

课程内容

在很多时候Spring的XML文件中也会有根据环境，部署方式变化的值。这些值最好也是不写死在XML中，而是从外部一个properties文件中读取：

1. 定义外部jdbc.properties

jdbc.username=hr

jdbc.password=hr

jdbc.url=jdbc:oracle:thin:@localhost:1521:xe

jdbc.driver=oracle.jdbc.OracleDriver

2. 引用外部properties

<bean class="org.springframework.beans.factory.config.PropertyPlaceholderConfigurer">

       <property name="location" value="classpath:day2/jdbc.properties"/>

</bean>

3. 引用properties中的key

<bean id="dataSource"class="org.springframework.jdbc.datasource.DriverManagerDataSource">

       <property name="url" value="${jdbc.url}"/>

       <property name="username" value="${jdbc.username}"/>

       <property name="password" value="${jdbc.password}"/>

       <property name="driverClassName" value="${jdbc.driver}"/>

</bean>

 

注意：如果properties文件与系统环境变量有命名冲突，PropertyPlaceholderConfigurer总是用properties文件中覆盖系统环境变量中的同名变量，在spring 3.1以后，采用了PropertySourcesPlaceholderConfigurer来实现更灵活的配置，它默认优先使用系统环境变量，如果设置了localOverride=true则优先使用properties中的变量

提高：PropertyPlaceholderConfigurer和CustomEditorConfigurer都实现了BeanFactoryPostProcessor接口，BeanFactoryPostProcessor接口会被spring容器自动调用，调用时机在BeanFactory被创建之后，Bean的实例被创建之前，用途是提供修改或新增Bean Definition的一个扩展点。
