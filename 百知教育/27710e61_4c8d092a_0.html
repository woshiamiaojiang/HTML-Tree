第1讲 MVC模式

课程内容

1. 什么是MVC模式

       先分别解释MVC的概念：

²  M(odel) 模型：是指数据以及操作数据的逻辑

²  V(iew) 视图：是指数据的展现方式

²  C(ontroller) 控制器：用于控制应用程序的流程，在模型和视图之间起到桥梁的作用

然后让学员对应刚做完的BB项目，指出哪些类对应模型、哪些类对应视图、哪些类对应控制器：

²  Entity和Service类对应模型

²  JSP对应视图

²  Servlet (Action)对应控制器，负责接收请求参数、调用业务逻辑、存储结果至作用域、转发至JSP视图

最后指出MVC是一种设计模式（实际上Web中的MVC模式是一种MVC变体被称为Model-2，这个可以不提）

 

2. MVC的优点

多个视图能够共享一个模型，同一个模型可以被不同的视图所重用，大大提高了代码的可重用性。例如：同一组城市数据，可以展现为下拉列表、也可以展现为树状结构。

控制器分离了模型和视图，让模型与视图保持相对独立，所以可以方便地改变应用程序的数据层和业务规。例如：可以将数据库从MySQL移植到Oracle，或者把RDBMS数据源改变为LDAP数据源，视图都会正确地显示。

总的来说，MVC的三个模块相互独立，改变其中一个不会影响到另两个。

      

3. MVC框架Struts 2

Struts2是一个实现了MVC模式的框架，它从另外两个MVC框架Struts和WebWork发展而来，并集成了两个框架的优点。

框架的意义在于

1.       提供基础、通用的操作，让程序员不必从头开发，提高开发效率

2.       让程序员的代码更为规范，必须按照框架的规范来编写代码，提高代码的可维护性

2.  配置前控制器StrutsPrepareAndExecuteFilter

打开web.xml文件添加如下配置：

 
 

  <filter>

 

      <filter-name>struts2</filter-name>

 

      <filter-class>
            org.apache.struts2.dispatcher.ng.filter.StrutsPrepareAndExecuteFilter
        </filter-class>

 

  </filter>

 

  <filter-mapping>

 

      <filter-name>struts2</filter-name>

 

      <url-pattern>/*</url-pattern>

 

  </filter-mapping>

 

前控制器的主要作用是提供一个统一入口，所有请求都先经过前控制器，由它再做进一步的分发处理。

3.  编写后控制器Action类

创建Action类，Action类就是一个POJO（Plain Old Java Object）

延伸概念：所谓POJO就是指一个普通java类，不需要实现特殊接口，也不必继承特定父类，它是相对于Servlet、EJB提出的一个概念。因为不必依赖于特定容器，因此容易被测试。

Action类需要有一个无参构造方法，控制器方法的名字任意，但不能带参数，并且返回值类型必须是String.

例如：

 
 

public class WelcomeAction { 

 

       public String abc() {

 

              System.out.println("进入了控制器方法");

 

              return null;

 

       }

 

}

 

注意：这里暂时不引入返回值的含义，先return null即可

4.  配置Action类，编写src/struts.xml

配置文件的名字和位置需要强调：src/struts.xml，大小写也是敏感的

使用MyEclipse XML文件创建向导来创建struts.xml

补充内容：时间充裕的话介绍Eclipse 中自定义XML Catalog配置DTD方法

上节的Action类配置如下：

 
 

    <!-- package 标签

 

                     name   是package的名字，要唯一

 

                     extends   表示继承另一个package的配置，固定写法

 

                     namespace   是地址栏路径的组成部分，必须以  "/" 开头

 

       -->

 

       <package name="p1" extends="struts-default" namespace="/a1">

 

              <!-- action 标签

 

             name 对应浏览器地址栏路径

 

                      class 对应java类名字

 

                      method 对应java类中的方法名

 

              -->

 

              <action name="hello" class="action.day1.WelcomeAction" method="abc">

 

              </action>

 

       </package>

6.  接收请求参数

两种方法：

1.        使用原始的HttpServletRequest对象来获取请求参数（不推荐）

HttpServletRequest request = ServletActionContext.getRequest();

request.getParameter("请求参数名");

2.        使用Action的属性来接收请求参数

a)        Action的属性名与请求参数名一致并提供get，set方法（区分大小写）

注意：

                        i.             验证请求参数其实是与set方法对应而非属性名

                      ii.             建议使用包装类型而非基本类型，这样可以用null值表示参数没有传递的情况

b)       Action的属性类型设置为需要的类型，Struts能够为常见类型做转换

注意：具体实施时，可以在HelloWorld例子中先讲最基本的，把复杂的放在后面讲

第3讲 访问Web三种作用域

课程内容

1.  存取作用域变量

向 request 作用域存：将结果存入Action属性

注意：原理需要到第三章讲完OGNL和值栈后解释

       向 session 作用域存：

              ActionContext.getContext().getSession().put("key", value);

       向 application 作用域存：

              ActionContext.getContext().getApplication().put("key", value);

 

2.  原理

带学员看ActionContext.getContext().getSession()的实际返回类型为SessionMap类型，教会学员如何连接到源码，并观察SessionMap中put方法和get方法的实现。用类似方式讲解ApplicatonMap的实现。

struts这样做的目的，主要是为了解除Action类与Servlet API直接的耦合。
