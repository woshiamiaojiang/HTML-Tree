第3讲 拦截器

课程内容

1.  自定义拦截器

 
 

public class Inter1 implements Interceptor {

 

       public String intercept(ActionInvocation ai) throws Exception {

 

              System.out.println("进入 拦截器1 ...");

 

              String   r = ai.invoke();

 

              System.out.println("拦截器1 执行完毕...");

 

              return r;

 

       }

 

}

 

要点：

1. 实现Interceptor接口

2. 在intercept方法内：

调用了ai.invoke()此拦截器放行，进入下一个拦截器，如果没有下一个拦截器，进入action的方法

没有调用ai.invoke(),表示请求被拦截

3. intercept方法的返回值表示，当请求被拦截时，要跳转到的视图名

2.  配置拦截器

 
 

<!-- 声明拦截器 -->

 

<interceptors>

 

       <interceptor name="i1" class="action.day5.Inter1"/>

 

       <interceptor name="i2" class="action.day5.Inter2"/>                    

 

</interceptors>

 

<action name="targetAction"  class="action.day5.IntercepterAction">

 

       <!-- 引用拦截器 -->

 

       <interceptor-ref name="i1"/>

 

       <interceptor-ref name="i2"/>

 

</action>

 

注意：进入拦截器的顺序与引用拦截器的顺序一致

3.  拦截器栈

所谓拦截器栈，就是指一组有顺序的拦截器，它可以用来简化拦截器的引用

 
 

<!-- 声明拦截器 -->

 

<interceptors>

 

       <interceptor name="i1" class="action.day5.Inter1"/>

 

       <interceptor name="i2" class="action.day5.Inter2"/>

 

       <!-- 定义拦截器栈 -->

 

       <interceptor-stack name="s1">

 

              <interceptor-ref name="i1"/>

 

              <interceptor-ref name="i2"/>

 

       </interceptor-stack>

 

</interceptors>

 

<action name="targetAction"  class="action.day5.IntercepterAction">

 

       <!-- 引用拦截器栈 -->

 

       <interceptor-ref name="s1"/>

 

</action>

 

还可以利用“默认拦截器引用”来进一步简化配置：

 
 

<default-interceptor-ref  name="s1"/>

 

这样可以省略每个Action中对拦截器栈的引用，但要注意：如果同时配置了Action中的拦截器引用和默认拦截器引用，则Action的拦截器引用会覆盖默认拦截器引用。以反例说明，如果配置了自己的默认拦截器栈，Action中利用属性接收请求参数的功能会失效，从而引出下一个知识点，系统拦截器。

4.  系统拦截器

带领学员查看struts-default.xml中系统定义的拦截器，并主要介绍常见系统拦截器的功能：

                               
 

拦截器名

  		 

功能简介

 
 

exception

  		 

处理异常，处理<exception-mapping>

 
 

i18n

  		 

处理国际化

 
 

chain

  		 

跳转方式为chain时，会用到此拦截器

 
 

fileUpload

  		 

处理文件上传

 
 

params

  		 

处理请求参数

 
 

validation     

  		 

调用validate方法

 
 

workflow

  		 

检查验证的错误集合是否为空，不为空跳转到input视图

 

在刚才出问题的拦截器栈中添加params拦截器，说明params拦截器的作用。

Struts将常见拦截器组合为一个名为defaultStack的拦截器栈，并作为默认拦截器引用。当用到自定义拦截器栈时，需要与defaultStack拦截器栈组合使用：

 
 

<!-- 声明拦截器 -->

 

<interceptors>

 

       <interceptor name="i1" class="action.day5.Inter1"/>

 

       <interceptor name="i2" class="action.day5.Inter2"/>

 

       <!-- 自定义拦截器栈 -->

 

       <interceptor-stack name="s1">

 

              <interceptor-ref name="i1"/>

 

              <interceptor-ref name="i2"/>

 

              <interceptor-ref name="defaultStack"/>

 

       </interceptor-stack>

 

</interceptors>

 

 

 

<default-interceptor-ref  name="s1"/>

 

 

5.  拦截器的应用——登录验证

 

 
 

public class LoginCheckInterceptor implements Interceptor{

 

       ...

 

       public String intercept(ActionInvocation ai) throws Exception {

 

              User   user = (User) ActionContext.getContext().getSession().get("user");

 

              if(user != null) { // 已登录，则放行

 

                     return ai.invoke();

 

              }   else { // 没有登录, 跳转到login的视图

 

                     return "login"; // 返回的字符串，代表视图名

 

              }

 

       }

 

}

 

 

 
 

<package name="homework"  extends="struts-default"  namespace="/homework">

 

       <interceptors>

 

        <!-- 定义登录拦截器 -->

 

              <interceptor name="logincheck" class="zpark.interceptor.LoginCheckInterceptor"/>

 

              <!-- 将登录拦截器与系统默认拦截器组组合为mystack -->

 

              <interceptor-stack name="mystack">

 

                     <interceptor-ref name="logincheck"/>

 

                     <interceptor-ref name="defaultStack"/>

 

              </interceptor-stack>

 

       </interceptors>

 

       <!-- 指定默认拦截器栈为mystack -->     

 

       <default-interceptor-ref name="mystack"/>            

 

       <global-results>

 

              <result name="login" type="redirect">/login.jsp</result>

 

       </global-results>

 

       <!-- 登录action 自己并不需要登录拦截器，所以仍引用defaultStack -->

 

       <action name="login" class="zpark.action.LoginAction" method="login">

 

              <interceptor-ref name="defaultStack"/>

 

              <result name="success" type="redirectAction">

 

                     <param name="actionName">findall</param>

 

              </result>

 

       </action>

 

       <!-- 其他action 引用默认的mystack -->

 

       <action name="findall" class="zpark.action.FindAllAction">

 

              <result name="success">/homework/findall.jsp</result>

 

       </action>

 

</package>

 

 

注意：登录验证功能也可以使用过滤器实现，但要注意将自定义过滤器的配置： <filter-mapping>的标签放在struts的Filter的<filter-mapping>标签之前，因为struts的Filter不会对请求放行。

7.  面向切面编程

像处理请求参数、处理异常、处理国际化，这些通用功能被集中到在拦截器中统一调用，每个拦截器处理一个功能，并能方便地自由组合这些功能，这种编程思想被称为面向切面编程AOP（Aspect Oriented Programming），它是对面向对象编程OOP（Obejct Oriented Programming）的一个补充。

所谓切面，就是正常流程中具有通用性质的代码被剥离出来，单独作为一个类存在，并在特定时间地点被调用。在Struts中，Action执行时用到的每个拦截器即可被看作一个切面。
