jsp简介

知识点

简介

JSP 全称 Java Server Pages，是servlet发展中的一个升级产品。与Servlet类似，运行在服务器内部，能够接受用户的请求，并产生动态的输出结果。

所有的JSP源文件都以 “.jsp”结尾；

Jsp源文件既可以保存在web应用的根目录下，也可以保存在web应用下的某个子目录中；

通过JSP可以将Java代码嵌入到html标签中，产生动态的输出结果。

第1个JSP

time.jsp

<%@ page pageEncoding="UTF-8"contentType="text/html"%>

<html>

    <head>

       <title>第1个JSP页面</title>

    </head>

    <body>

       <h1><font color="green">

       <%= new java.util.Date()%>

       </font></h1>

    </body>

</html>

如果time.jsp保存在web应用的根下：

http://localhost:8989/jsp_day1/time.jsp

如果time.jsp保存在web应用的某个子目录下面：

http://localhost:8989/jsp_day1/subdir/time.jsp

jsp的特征

ü  不需要编译，就可以直接运行（如果jsp有编译错误，会显示在浏览器上）

ü  代码修改之后，不需要重新启动服务器

ü  不需要配置，浏览器可以向访问HTML文件一样访问JSP

Jsp和servlet的关系

JSP更擅长于显示结果，Servlet更擅长于处理业务

ü  JSP是将java代码嵌入到html中，html是jsp得主体

ü  Servlet是将HTML嵌入到java中，java是Servlet的主体

JSP的执行过程

知识点

1、翻译 .jsp  -->  .java

              翻译规则：

              <%= %>   --  service方法中的 out.println()

              <%! %>  --  servlet类的成员变量和成员方法

              <%  %> --   service方法中的 普通代码

              html内容 --   service方法中的 out.write()

       2、编译  .java --> .class

       3、运行 .class

注意：在JSP执行时，第1次执行慢，以后快。

原因：第1次执行要经过编译、编译和执行3个阶段；以后次执行，不需要编译、编译，直接执行已经编译好的class文件。
