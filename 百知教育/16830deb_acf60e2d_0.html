应用程序的体系结构

知识点

C/S结构

Ø  即client/server结构的应用程序，使用一个桌面程序作为客户端，使用自定义的应用层协议与服务器通信。

Ø  缺点：

client需要安装特定的软件；

维护和升级麻烦；

B/S结构

Ø  即Browser/Server结构的应用程序，使用浏览器作为客户端，用户界面是一张网页 使用http协议与服务器通信。

Ø  优点：

在用户的电脑上无需安装任何专用的客户端程序。

所有资源都集中在服务器上，容易维护和升级。

浏览器与服务器之间使用http协议通信，与互联网完美结合。

随着web UI技术的发展，网页具有了与桌面应用相同的用户体验

Ø  常见的B/S结构应用程序：

网银

电子商务平台

搜索引擎

Tomcat服务器

知识点

静态页面

Ø  工作原理

1、输入URL，通过互联网与WEB服务器建立网络连接

2、WEB服务器从URL中获得HTML文件名称，读取文件的内容

3、WEB服务器将读取的文件内容通过网络传回浏览器

4、浏览器将网页内容显示给用户

Ø  不足之处

静态页面中的信息是在HTML中提前编写好的

企业级web应用的页面内容要根据运算动态显示。

因此，B/S结构的企业级应用程序必须让网页 动起来

动态页面

Ø  工作原理

1、输入URL，通过互联网与WEB服务器建立网络连接

2、WEB服务器接受到请求后调用一段Java程序

3、WEB服务器将Java程序的输出结果传输给客户端

4、浏览器将程序的输出结果显示在网页上

第1个Servlet

知识点

什么是servlet

Servlet是运行在WEB服务器中的一小段Java程序，它能够通过WEB服务器接收并处理浏览器发送的http请求, 能够通过WEB服务器将动态生成的结果应答给客户端，从而实现动态网页的功能。

由于web服务器为Servlet的运行提供了环境，所以web服务器也被称作web容器,或应用服务器，如Tomcat，JBOSS等等。

Servlet不会主动执行，必须由WEB服务器调用

Servlet的开发步骤

案例：实现在网页上动态显示当前日期时间

步骤1、写代码

三种方式：

       1) implements Servlet  必须重写所有的方法

       2) extends GenericServelt 重写service方法，与http协议无关

       3) extends HttpServlet 重写service方法，与http协议相关

           public void service(HttpServletRequest request, HttpServletResponse response)

                  throws ServletExcetpin, IOException{

              //设置响应类型

              response.setContentType("text/html");

             

              //获取输出流

              PrintWriter out = response.getWriter();

             

              //使用输出流

              out.println("<html>");

              out.println("<head><title>这是第一个Servlet</title></head>");

              out.println("<body>");

              java.util.Date now = new java.util.Date();

              out.println(now);

              out.println("</body>");

              out.println("</html>");

              out.flush();

           }

注意：必须在java project中，导入 tomcat安装根目录/lib/servlet-api.jar。

    步骤2、编译

MyEclipse自动编译

    步骤3、部署

       1) 根据JavaEE规范，创建目录结构

           tomcat安装根目录/webapps/myweb(web应用名称)

              |+ WEB-INF 必须

                  |-web.xml 必须，有且只能有1个

                     (从ROOT目录中copy，保留第1行和web-apps根标签)

                  |+classes 存放class文件

                  |+lib     存放jar包

              |+css

              |+images

              |+user

              |- .html .css .gif

       2) copy 编译后的类文件（连同包结构）到 tomcat根目录/webapps/myweb/WEB-INF/classes。

    步骤4、配置

       必须在web.xml中配置：

       <servlet> <!-- 配置servlet的相关信息 -->

           <servlet-name></servlet-name><!--别名,唯一 -->

           <servlet-class></servlet-class><!-- servlet全类名 -->

       </servlet>

       <servlet-mapping>   <!-- 把class映射成url可访问的资源 -->

           <servlet-name></servlet-name><!--别名,唯一 -->

           <url-pattern></url-pattern><!--请求servlet时url的地址格式，必须以/开头 -->

       </servlet-mapping>

    步骤5、测试运行

       http://ip:port/web应用名称/url-pattern

       http://localhost:8989/myweb/hello

常见问题

2、部署时copy类文件，常见问题：a、把源代码copy过来；

b、copy时，不带包结构。

3、404 请求资源无效

    检查URL地址栏中的字符

4、405

5、500

    检查java源码运行是否有异常；

    检查web.xml中<servlet-class>全类名是否写错。

response.setCharacterEncoding("gbk");
