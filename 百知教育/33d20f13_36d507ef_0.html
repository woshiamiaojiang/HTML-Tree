2.        异步请求 与 同步请求
同步请求 (地址栏 超级链接 表单 location.href )
问题:用户的交互体验差
现象:同步请求响应给client是一张新的页面
     同步请求必须等待server的响应,才可以进行后续操作,否则操作无意义
异步请求 (XMLHttpRequest)
好处:用户的交互体验好
现象:异步请求响应给client不是一张新的页面,而是页面的一个局部(字符串)
     异步请求无需等待server的响应,可以直接进行后续操作

3.        异步请求的编码思路分析
异步请求对象 XMLHttpRequest 是一个javascript对象 存在浏览器差异,把简称xhr.

a)       创建xhr对象
Webkit: var xhr = new XMLHttpRequest();
IE:    var xhr = new ActiveXObject(“Microsoft.XMLHTTP”);
if(window.XMLHttpRequest){
   xhr = new XMLHttpRequest();
}else{
   xhr = new ActiveXObject(“Microsoft.XMLHTTP”);
}

b)       xhr对象发送请求 并 传递参数
xhr.open(“method”,”url”);作用: 发送请求
   例如: xhr.open(“get”,”/ajax/checkAjaxServlet”);
xhr.send() 作用:往服务器端传递参数

c)        xhr处理响应
btn.onclick = function(){

}
xhr.readyState
0 --- 4
readyState == 0 说明 xhr对象未创建
readyState == 1 说明 xhr发送请求
readyState == 2 说明 说明服务器端得响应到达client
readyState == 3 说明 服务器端得响应开始解析
readyState == 4 说明 响应完毕
xhr.status
200 正常
404 路径问题
500 服务器端程序错误

xhr.onreadystatechange = function(){
   if(xhr.readyState==4 && xhr.status==200){
        处理
   }
}
