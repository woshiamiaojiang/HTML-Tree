2.    如何在插入记录时生成主键

方法1. 通过max求得下一个数

select case when max(a) is null then 1 else max(a)+1 end from x;

隐患：

l  当数据量较多时，每次都运行max会影响效率

l  单考虑上述查询，当查询并发运行时会出现主键重复现象

改进办法：仍使用上SQL获取主键值，插入时如出现主键重复异常，捕获此异常并重新插入

 

方法2. 通过SYS_GUID()产生主键

产生128bit的数字，如果仅希望主键产生一个唯一值的情况下，较为适合

 

方法3. 使用单独表维护主键

首先创建以下表，并插入一条初始记录维护x表的主键

create table idGenerator(

       table_name varchar2(20) primary key,

       curr_val   number(20) not null

);

insert into idGenerator values ('x', 1);

commit;

以后产生主键时使用：

select curr_val from idGenerator where table_name = 'x' for update;

update idGenerator set curr_val = curr_val + 1 where table_name = 'x';

commit;

第一条语句加for update是为了防止并发时产生重复主键，第二条更新是用来产生下一个值

3.    Oracle中可以使用序列来解决主键生成问题。

语法：create sequence 序列名;

 

-- start with 设置序列取值的开始位置

create sequence seq_1 start with 300001;

 

-- cache 5 从序列中取得下5个值放置在缓存,默认为20，主要目的是提高效率

create sequence seq_1 cache 5;

 

使用方式：

create sequence product_yihang_seq start with 100001;

第6讲 视图

1.    视图(View)的概念及语法：

命名的查询，虚拟表

create view 视图名 as 查询语句

 

2.    视图的作用

作用1：用于简化复杂的查询

例：对于一些复杂的查询，使用视图：

drop view orderView;

create view orderView as

select c.custid, c.name, o.orderdate, o.total, i.qty, p.descrip

  from customer c left join ord o on (c.custid = o.custid)

    left join item i on (o.ordid = i.ordid)

    left join product p on (i.prodid = p.prodid);

-- 可以简化为：

select * from orderview where custid = 100;

 

作用2：隐藏基表结构

允许其他用户查询视图，但不能让其他用户直接访问基表

create view empView as

select empno,ename, job, hiredate from emp;

 

grant select on empview to zjkcksd0901;

grant insert on empview to zjkcksd0901;

 

3.    注意事项：

注意1：视图不保存查询结果，是一个虚拟的表

注意2：视图是否能提高查询效率（不能）

注意3：也可以对视图增、删、改, 但一般不这样用,而且限制很多

(要对视图增、删、改，视图只能对应一个基表)

注意4：select * from user_views; --查询视图信息

第7讲 索引

1.    索引(Index)的概念

目的：提高查询的效率

 

2.    什么时候使用索引?

    1) 在经常查询的列上建立索引

    2) 在大表上(数据量大)

    3) 索引是用来查询少量信息的(占数据总量的30%以下)

    例：假如用户经常执行下列SQL语句

    select * from emp where ename = ?;

而且emp表中记录相当多，并且员工姓名重复的情况很少，这时应当在emp表的ename列上建立索引

 

3.    语法：

create index emp_ename_idx on emp(ename);

 

4.    注意事项：

注意1.索引不需要显式调用，数据库会自动决定什么时候应该应用索引

 

注意2.索引提高查询效率的原理：

    1) 采用索引段来保存了排序之后的结果

    2) 查询语句首先从索引段快速查询到rowid

    3) 根据rowid的值到数据段找到对应数据

回忆rowid伪列: 能够唯一标识表里的某一条记录，能够使用rowid最快速定位到记录

oracle在索引段数据的排序结构B(alance) * 树

 

注意3.

    1) 索引要占用额外的存储空间，所以不是建的越多越好

    2) 索引会影响插入和删除的效率

    3) 建立主键，会自动在主键列上建立唯一索引

    如何建立唯一索引

           create unique index emp_ename_idx on emp(ename);

    4) 建议外键列上建立索引，会提高表连接的效率

    5) 对于下列情况不会应用ename列的索引

select * from emp where lower(ename) = 'smith';

    解决方法

           create index emp_ename_idx2 on emp(lower(ename));
