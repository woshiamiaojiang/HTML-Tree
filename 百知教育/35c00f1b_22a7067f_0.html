jstl

知识点

由apache的jakarta小组开发的运行在JSP上的一组标签库

可以实现流程控制、URL操作等功能

通过使用JSTL,可以替代java标签，大大简化JSP代码

使用前的准备：

}  确保在web 项目中导入了jstl.jar 和 standard.jar

这两个jar包提供了实现jstl标签功能的java代码

}  在使用jstl的jsp中导入标签库

<%@taglib uri=“http://java.sun.com/jsp/jstl/core” prefix=“c”%>

uri ：标签库的唯一标识,为了保证唯一性，通常使用一个url或者一个email地址。

prefix : 标签库前缀，用于区分不同标签库中的同名标签

通用标签

1、设置命名属性：

<c:set var="" value="" scope=""/>

              var: 要设置的命名属性的名称

              value: 要设置的命名属性的值，可以是常量、<%=%>、${}。

              scope: 所在的作用域，可取值-page(默认)|request|session|application

       如，

往 pageContext 中设置一个命名属性：<c:set  var=“test” value=“123456”/>

往 reqeust 中设置一个命名属性：<c:set var=“a” value=“1” scope=“request”/>

将标签体的输出结果作为a的值：

<c:set var=“a”>

 <h1><font color=‘red’>${1+1}</font></h1>

</c:set>

该标签的标签体内容不会显示在网页上，而是作为命名属性的a的值保存起来。我们可以输出a来显示标签体 ${ a }或${pageScope.a}

2、删除命名属性

       <c:remove var="" scope=""/>

              var: 要删除的命名属性的名称

              scope: 所在的作用域，可取值-page|request|session|application

                            注意：默认删除所有作用域中的同名命名属性。

如，

删除所有作用域中的命名属性a：<c:remove var=“a” />

移除session中的命名属性a：<c:remove var=“a” scope=“session”/>

3、输出命名属性的值

       <c:out value="" escapeXml=“true|false”/>

              value: 要显示的内容，可以是常量、<%=%>、${}。

              escapeXml：是否直接显示xml中的特殊字符 < , > , & 等

如，

输出命名属性a的值：<c:out value=“${a}”/>

输出命名属性a的值，如果value的值为空，默认输出123：<c:out value=“${a}   default=“123”/>

输出内容中直接显示 2>1： <c:out value=“2>1” escapeXml=“true” />

分支标签

1、单分支 c:if

<c:if test=“${ a > 10}”>

     body

</c:if>

相当于java中的if语句，如果test属性指定的布尔表达式为真时，标签题内的代码才会执行

如：<c:set var=“name” value=“${param.name}”/>

<c:if test=“${ not empty name}”>

       <h1>your name is ${name}</h1>

</c:if>

注意：JSTL是没提供对应的else标签。可以通过使用两个相反的条件来表示。

如，<c:set var=“name” value=“${param.name}”/>

<c:if test=“${not empty name}”>

       <h1>Hello ,your name is ${name}</h1>

</c:if>

<c:if test=“${empty name}”>

       <h1>What’s your name ?</h1>

</c:if>

2、多分支 c:choose

该标签，类似于Java中的switch … case … 语句。

<c:choose>

        <c:when test=“${c1}”>

              body 1

     </c:when>

    <c:when test=“${c2}”>

          body 2

    </c:when>

    <c:when test=“${c3}”>

             body 3

    </c:when>

    <c:otherwise>

          body 4

    </c:otherwise>

</c:choose>

 

“分支”的综合示例代码：

<%@ taglib uri="http://java.sun.com/jsp/jstl/core"prefix="c" %>

<%

        //测试数据

        int num = 5;

        pageContext.setAttribute("MyNum",num);

    %>

<c:if test="<%=(num==5) %>">

    <font color="red">OK-111!</font>

</c:if>

<br/>

<c:if test="${pageScope.MyNum == 5 }">

        <font color="red">OK-222!</font>

</c:if>

<c:if test="${pageScope.MyNum != 5 }">

        <font color="red">error!</font>

</c:if>

<hr/>

<c:choose>

        <c:when test="${pageScope.MyNum == 5 }"><font color="red">OK-333!</font></c:when>

        <c:when test="${pageScope.MyNum == 6 }"><font color="blue">error</font></c:when>

        <c:otherwise>nonono......</c:otherwise>

</c:choose>

迭代标签

功能类似于 java 中的 for 循环语句。

1、固定次数的循环

<c:forEach  begin=“1”   end=“5” step=”” var=”” >

              <h2>Hello world</h2>

</c:forEach>

上面循环语句，从1开始，到小于等于5 结束 ，上例中的标签体一共被执行6次

       var: 循环变量的名称，在JSTL标签内部使用该变量时，必须使用EL。

       begin: 初值

       end: 结束值

       step: 递增量

2、对list和map集合迭代

<c:forEach items="" var="" varStatus="">x</c:forEach>

              items: 要遍历的集合内容，可以使用<%= %>或${}

              var: 每遍历1次取出的集合元素，所存放的变量，使用时必须用EL。

                            如果是map的集合元素，则需要使用${m.key}取出key; ${m.value} 取出value。

              varStatus: 迭代状态对象，常用的属性-

                                   count: 迭代的次数，1,2,3....

                                   index: 迭代的索引，从0开始（对有下标的遍历有意义）

       如，

<c:forEach var=“student” items=“${stuList}”>

              <tr>

                 <td>${student.name}</td>

            <td>${student.age}</td>

        </tr>

</c:forEach>

<c:forEach var=“m” items=“${myMap}”>

       ${m.key.name}，${m.value.name}，${m.value.age} <br/>

</c:forEach>

 

示例：

<%

        //测试数据

        // list集合

        List<User> list = new ArrayList<User>();

        list.add(new User("zs","111"));

        list.add(new User("ls","222"));

        ${i } hello <br/>

</c:forEach>

<hr/>

    ${u.username } , ${u.password } <br/>

</c:forEach><br/>

        ${u.username } , ${u.password } -- ${s.count } -- ${s.index }<br/>

</c:forEach>

<hr/>

</c:forEach>

<%

    // 测试数据

    int pageNo = 0;

    String strPage = request.getParameter("p");

    if (strPage == null)

       pageNo = 1;

    else

       pageNo = Integer.parseInt(strPage);

    int maxPage = 8;

    pageContext.setAttribute("curPage", pageNo);

    pageContext.setAttribute("totalPage", maxPage);

%>

当前第 ${curPage } 页  共${totalPage } 页

<c:if test="${curPage == 1 }"> 

    首页

</c:if>

<c:if test="${curPage > 1 }">  

    <a href="${pageContext.request.contextPath }/splitPage.jsp?p=1">首页</a>

</c:if>

<c:if test="${curPage == 1 }"> 

    上一页 

</c:if>

<c:if test="${curPage > 1 }"> 

    <a href="${pageContext.request.contextPath }/splitPage.jsp?p=${curPage-1}">上一页</a>

</c:if>

<c:forEach var="i" begin="1" end="${totalPage }" step="1">

    <c:if test="${curPage == i }">

       ${i }

    </c:if>

    <c:if test="${curPage != i }">

    <a href="${pageContext.request.contextPath }/splitPage.jsp?p=${i}">${i }</a>

    </c:if>

</c:forEach>

 

<c:if test="${curPage == totalPage }"> 

    下一页

</c:if>

<c:if test="${curPage < totalPage }"> 

    <a href="${pageContext.request.contextPath }/splitPage.jsp?p=${curPage+1}">下一页</a>

</c:if>

<c:if test="${curPage == totalPage }"> 

    尾页

</c:if>

<c:if test="${curPage < totalPage }"> 

    <a href="${pageContext.request.contextPath }/splitPage.jsp?p=${totalPage}">尾页</a>

</c:if>
