创建表的语法：

          columnName columnType [CONSTRAINT constraintName]constraintType,

其中tableName（表名），columnName（列名），columnType（数据类型），constriaintName（约束名），constraintType（约束类型） 。

oracle中的数据类型 ：

1.        字符类型：Char ,Varchar2

在使用char或varchar2时必须指定字符串的最大长度，如char(10 )、 varchar2(30) 。

如果向表中插入长度超过最大长度的数据，将会发生错误。

两种char的区别在于，char 为定长字符类型，varchar2变长字符类型。所谓定长字符串是指数据的长度是固定的，即使存入的数据小于最大长度，也会占用同样的存储空间，多于的长度用空格补齐。变长字符串的含义是列的长度随着数据的实际长度进行调整，只要不超过最大值就可以，空闲长度不需要空格补齐。

 

比如：

   Create table t(

       C1 char( 10 ) ,

       C2  varchar2( 10 )

   )

 

   Insert into t( c1 , c2 ) values( ‘liucy’ , ‘liucy’ ) ;

我们来观察两个列的情况：

Select * from t ;

 

Select length( c1 ) as c1_length , length( c2 ) as c2_length from t ;

我们可以才看出，c1的实际长度比c2多出5个，那么为什么在查询c1,c2的值的时候无法看到多出的五个字符呢？适用于在数据库中，空格属于不显示字符。所以无法直接看见，我们可以在查询中实用替换函数，将所有的空格替换为”*”号，就可以看出问题所在了。

select replace( c1 , ' ' , '*' ) as c1 from t ;

 

从上图中我们就可看出为什么c1比c5的实际长度多五个了。

7)          Foreign Key 外键约束：用来定义表与表之间的关系并约束子表中关系列的取值，表和表之间的关系大家可以回忆一下我们学习链接查询时的查询条件。在Employees表中只保留了所属部门的ID,部门的实际信息保存在部门表里，如果我们想在查询员工信息的同时也获得部门信息的话，那么就需要拿着员工表中的部门ID到部门表中查找匹配的记录。在这种情况下，对于Employees表来讲，department_id 这一列就称作外键，因为这一列指向了另外一张表的主键。同时，部门表称作主表，员工表称作子表。

8)          组合约束

 

有时候，我们需要将一个约束加载两个锅多个列上，这种约束称作组合约束,现有成绩表结构如下：

 

create table scores

 

(

 

   studentId number(4) not   null,

 

   courseName varchar2(10)   not null,

 

   score number(3) not null

 

);

 

如何添加约束，保证学生每门科目只有一个成绩

 

例如不能有

 

studentId        courseName    score

 

1001              java               20

 

1001              oracle             49

 

1002              java               30

 

1001              java               90

 

我们不难发现，学生的学号可以重复，课程的名称也可以重复，不能重复的是学号+课程。这种约束作用于两列或多列，称之为联合（组合）约束，比如说上例可以这样添加一个唯一约束：

 

create table scores

 

(

 

   studentId number(4) not   null,

 

   courseName varchar2(10)   not null,

 

   score number(3) not null,

 

   unique(studentId, courseName)

 

);

 

约束的写法是另起一行（注意,号），说明约束的类型和作用列。

 

其实上例也可以使用主键约束

 

create table scores

 

(

 

   studentId number(4) not   null,

 

   courseName varchar2(10)   not null,

 

   score number(3) not null,

 

   primary key(studentId, courseName)

 

);

 

在分类上也把这种约束叫做表级约束，前面形式的约束叫做列级约束。所有的列级约束（除NOT NULL之外）都可以表示为表级约束（反之不行），例如：

 

create table students

 

(

 

   studentId number(4)   primary key,

 

   studentName varchar2(10)   not null

 

);

 

       可替换为

 

       create   table students

 

(

 

   studentId number(4),

 

   studentName varchar2(10)   not null,

 

   primary key (studentId)

 

);

 

对于外键约束来说，表级约束写法稍有区别，例如，前例中orders表要使用表级约束的写法应当这样：

 

create table orders

 

(

 

  orderId number(6)   constraint orders_orderid_pk primary key,

 

  orderdate date constraint   orders_orderdate_nn not null,

 

  state varchar2(20) default   sysdate constraint orders_state_nn not null constraint

 

orders_state_ck check (state in   ('Y','N')),

 

  totalPrice number(10,2)   constraint orders_totalprice_nn not null constraint

 

orders_totalprice_ck check (totalPrice   > 0.0),

 

  username varchar2(20)   constraint orders_username_nn not null,

 

  constraint   orders_username_fk foreign key(username)

 

references users (username)

 

);

 

 

 

 

 

 

3.    根据已有表创建新表

我们还可以根据已有的表来创建新表，这样已存在的表中的

 

语法：

CREATE TABLE tableName AS SELECT … FROM …

或者：

CREATE TABLE tableName(

       columnName[CONSTRAINT constraintName] constraintType,

       …    ,

       …    ,

       …  

) AS SELECT … FROM …

 

 

 

需要注意的是，表复制其实涉及以下三个操作

1)       根据查询的列的信息创建新表

2)       根据查询的行的信息为新表填充数据

3)       如使用第一种语法，旧表的not null约束会被复制到新表，其他约束不会复制

 

例：

简单复制EMP表的结构和数据

create table newemp as select * from emp;

 

新表结构和原表不同的情况

create table newemp as select count(*) num, deptno from emp group by deptno;

 

只复制表结构，不复制数据

create table newemp as select * from emp where 1 = 2;

 

自定义复制EMP表结构和数据

create table newemp (

       empno constraint newemp_empno_pk primary key,

       ename constraint newemp_ename_nn not null,

       deptno constraint newemp_deptno_nn not null

) as select empno,ename,deptno from emp;

这种语法要求create部分与select部分的列数目一致，并不能直接建立外键约束

第2讲 DML命令

1.    INSERT

语法：

第一种，给出所有列的名称和值

INSERT INTO tableName (columnName1,columnName2, …)

VALUES (columnValue1, columnValue2, …);

      

       第二种，给出部分列的名称和值。

名称和值必须包括所有主键、不为空的列（除了指定了默认值的）

INSERT INTO tableName (columnName1,columnName2, …)

VALUES (columnValue1, columnValue2, …);

 

第三种，只给出值，必须包括所有列的值

INSERT INTO tableName VALUES (columnValue1,columnValue2, …);

 

第四种，将查询结果作为多行插入

INSERT INTO tableName  SELECT …;

例：

create table newemp as select * from emp where 1 = 2;

insert into newemp select * from emp where deptno = 10;

它的用途是一次可以插入多条记录

限制：

       插入的数据一般来源于旧表

       插入新表的列不一定跟旧表的列一致，但数据类型必须符合，如：

       insert into newemp (empno, deptno, sal) select empno, deptno, comm from emp;

2.    UPDATE

语法：

UPDATE tableName SET columnName1= columnValue1, columnName2= columnValue2, …

       WHERE … ;

细节：更新操作是怎样工作的，先介绍一个系统函数dbms_random.random()

例：求10000以内任意整数

select mod(abs(dbms_random.random()),10000) from dual;

再使用该值更新员工表的工资列：

update emp set sal = mod(abs(dbms_random.random()),10000);

注意更新操作也是逐行进行的。

 

3.    DELETE

语法：

DELETE FROM tableName WHERE… ;

注意：update和delete操作需要考虑是否增加where条件

删除记录时需要注意，必须先删除主表记录，才能删除从表记录，如：

delete from dept where deptno = 10;    -- 违反完整约束条件

只能够：

delete from emp where deptno = 10;

delete from dept where deptno = 10;
