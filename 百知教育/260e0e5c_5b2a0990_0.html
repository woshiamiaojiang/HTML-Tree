过滤器

知识点

如果一个Servlet配置了过滤器,那么请求在到达Servlet之前先要经过过滤器的处理。

自定义过滤器

1、写代码

       implements Filter   重写接口中的所有方法

              // 过滤处理

              public void doFilter(ServletRequest request,ServletResponse response, FilterChain chain) throws ServletException,IOException{

                     // 要过滤处理的功能代码

                     //....

                     // 把request/response继续往后传递

                     // 如果后面还有过滤器，则到下一过滤器执行；

                     // 如果没有过滤器，则直接到web资源

                     chain.doFilter(request, response);

              }

2、配置 web.xml

       <filter>

              <filter-name>别名</filter-name>

              <filter-class>包名.类名</filter-class>

       </filter>

       <filter-mapping>

              <filter-name>别名</filter-name>

              <!-- 指定该过滤器需要过滤处理的资源格式 -->

              <!-- 该web应用根目录/bank下的所有资源需要过滤 -->

              <url-pattern>/bank/*</url-pattern>

              <!-- 该web应用根目录/user下的所有资源需要过滤 -->

              <url-pattern>/user/*</url-pattern>

              <!-- 该web应用中的所有资源需要过滤 -->

              <url-pattern>/*</url-pattern>

       </filter-mapping>

举例：Filter1.java 和 TestServlet1.java

       public void doFilter(ServletRequest request, ServletResponse response,FilterChain chain) throws IOException, ServletException {

              System.out.println("in filter1---------begin----------");

              //把request/response继续往后传递

              chain.doFilter(request, response);

              System.out.println("in filter1---------end----------");

       }

配置：

       <filter>

              <filter-name>Filter1</filter-name>

              <filter-class>com.zpark.filter.Filter1</filter-class>

       </filter>

       <filter-mapping>

              <filter-name>Filter1</filter-name>

              <url-pattern>/*</url-pattern>

       </filter-mapping>

开发中的应用

1、设置中文字符集

功能代码：request.setCharacterEncoding(“UTF-8”);

                     response.setCharacterEncoding(“UTF-8”);

配置：

       <filter>

              <filter-name>EncodingFilter</filter-name>

              <filter-class>com.zpark.filter.EncodingFilter</filter-class>

       </filter>

       <filter-mapping>

              <filter-name>EncodingFilter</filter-name>

              <url-pattern>/*</url-pattern>

       </filter-mapping>

2、强制用户登录

功能代码：HttpServletRequest req = (HttpServletRequest) request;

              HttpServletResponse res = (HttpServletResponse) response;

              //功能代码

              HttpSession session = req.getSession(true);

              Object obj = session.getAttribute("LoginUser");

              if (obj == null) {  //没登录,回登录页

                     res.sendRedirect("/servlet_day5/login.html");

                     return;

              }  else {  //登录

                     chain.doFilter(request, response);

              }

       配置信息注意事项：

       a、区分该项目中，需要过滤和不需要过滤的资源

              把需要过滤的资源统一放到一个目录中

              servlet: 增删改查相关的servlet，在原url-pattern前添加目录结构，如 /bank/xxx

              页面html/jsp：增删改查相关的页面，移到新文件夹中，如 module

       b、配置

              <filter>

                     <filter-name>AuthFilter</filter-name>

                     <filter-class>com.zpark.filter.AuthFilter</filter-class>

              </filter>

              <filter-mapping>

                     <filter-name>AuthFilter</filter-name>

                     <url-pattern>/bank/*</url-pattern>

                     <url-pattern>/module/*</url-pattern>

              </filter-mapping>
