第3讲 整合Struts2

1.  将Action类交给Spring管理      

<bean id="findUserAction"class="day4.action.FindUserAction" scope="prototype">

       <property name="userService" ref="userService"></property>

</bean>

跟其它bean的定义以及依赖关系注入是类似的，唯一要注意的是scope必须是prototype，因为Struts2中每次请求必须创建新的Action实例

struts.xml中的Action配置则修改如下：

<package name="pk1"extends="struts-default"namespace="/user">

       <action name="findUser" class="findUserAction">

              <result name="success">/index.jsp</result>

       </action>

</package>

其中<action>中的class由类名改为spring中的bean的id

2.  添加struts2-spring-plugin.jar

这算是Struts提供的一个插件包，它的主要作用是找到spring容器，按<action>标签中的class值（bean id）到spring容器中找到Action类，让Action实例的创建由Struts改为Spring。

3.  添加spring提供的ContextLoaderListener

这是一个监听器，实现了ServletContextListener接口，会在应用程序启动和停止时被调用，用来创建和销毁spring容器，在spring容器创建成功后，该Listener会将spring容器放入application作用域，供其它代码访问：

      <listener>

              <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>

</listener>

它默认读取一个名为/WEB-INF/applicationContext.xml的spring配置文件，如果需要自定义配置文件位置，添加如下初始化参数：

<context-param>

              <param-name>contextConfigLocation</param-name>

              <param-value>classpath:day4/transaction.xml</param-value>

       </context-param>

 

做完如上配置，与Struts的整合就OK了。

提高：其实第一步可以不做，仍然让struts自己创建Action对象：

<package name="pk1"extends="struts-default"namespace="/user">

       <action name="findUser" class="day4.action.FindUserAction">

              <result name="success">/index.jsp</result>

       </action>

</package>

但要注意Action所依赖的Service的set方法要与spring容器的service bean的id一致（set方法去掉set后第一个字符变为小写==bean id）：

public classFindUserAction { 

       private UserService userService;

       public void setUserService(UserService userService) {

              this.userService = userService;

       }

}

struts2-spring-plugin.jar插件会在内部调用spring的AutowireCapableBeanFactory相关方法来完成属性注入操作。

4.  OpenSessionInViewFilter

回忆延迟加载异常的解决方法：

思路1： 在Hibernate Session关闭前，提前查询数据

²  关系属性上加lazy=false

²  Hibernate.initialize(user.getContacts());

²  用inner join fetch 连接抓取查询

Spring又为我们提供了另一种思路：

思路2：让session关闭推迟

原本Hibernate Session是在每个Service业务方法调用结束后，事务结束时关闭的。现在让Hibernate Session的打开时间提前，关闭时间推迟。具体就是在请求进入OpenSessionInViewFilter过滤器时打开，在响应返回时关闭。

过滤器的配置如下：

<filter>

       <filter-name>osiv</filter-name>            <filter-class>org.springframework.orm.hibernate3.support.OpenSessionInViewFilter</filter-class>

</filter>

<filter-mapping>

       <filter-name>osiv</filter-name>

       <url-pattern>/*</url-pattern>

</filter-mapping>

 

注意三点：

一．此过滤器的filter-mapping必须位于struts2的filter-mapping之前，否则请求到达不了

二．该过滤器仅是为了解决延迟加载而产生，因此只应当配置过滤那些查询action，像上面配置/*其实是非常不恰当的，这样即使请求jsp，html这些资源时，也会打开关闭Hibernate Session，得不偿失

三．在一些特殊场合下，会出现意想不到的错误

场景1：更新商品(Product)，它有一个关系属性类别(Category)

更新时，因为只需要类别id，所以类别对象没有从数据库查询，使用的是游离态类别对象

更新Action代码大致如下：

Product product = ... ; // 包括其中的Category（仅含id）都是根据请求参数由struts封装。

       productService.update(product); // 更新商品

list = productService.findAll(); // list也是Action的属性，本意是在更新后再次查询所有商品仍跳转回商品列表视图

问题来了，在返回商品列表后，发现商品的名称不见了。

分析：因为用了OpenSessionInViewFilter过滤器，因此更新和查询操作使用的同一个session，更新时，将该商品对象的状态变为了持久态，纳入session一级缓存管理；接下来的查询操作，在将结果集封装为商品对象时，在一级缓存中发现已经存在了该商品对象，所以不会生成新的商品对象，该商品的所有属性（包括其类别关系属性）都会使用内存中已存在的；最后到页面显示时，其他商品对象的类别都是代理对象，因此引发查询数据库操作，获得了正确的值，但刚才商品的类别是读的内存中的，仅有id值。

解决：

方法a，对于类别关系属性，使用其持久态对象，也就是更新前再查一遍类别（不太好）

方法b，更新操作和查询操作分开，用不同的Action（可以再让OpenSessionInViewFilter仅过滤查询Action）

 

场景2：更新用户

Action代码如下：

list = userService.findAll();    

       userService.update(user);

在执行到更新时会出现异常：

       a different object with the same identifier value was already associated with the session

 

分析：这个Action的本意也是更新完毕后，查询所有用户，调回用户列表页面。因为是同一session，查询方法以及将该用户对象查询出来变为持久态，而更新update方法也是要将该用户对象变为持久态，因此出现了上面的异常。

解决：

方法a：调整代码顺序，先更新，再查询（不推荐，会有潜在的问题，类似于场景1的描述）

方法b：将update方法改为merge方法

方法c：更新操作和查询操作分开，用不同的Action （推荐）

可以看到：使用了osiv过滤器之后， 正确的做法是让Action操作相互独立（session的生存变长后， 需要对Hibernate的对象状态有深刻了解，才能避免犯错）

 

第4讲 整合JSON

1.  当json转换遇到hibernate关系属性和代理对象

对于Gson,不能自动转换Hibernate中的代理对象，需要注册新的代理对象转换器：

public classHibernateProxyTypeAdapter extendsTypeAdapter<HibernateProxy> {

 

    public static final TypeAdapterFactory FACTORY = new TypeAdapterFactory() {

        @SuppressWarnings("unchecked")

        public <T> TypeAdapter<T> create(Gson gson, TypeToken<T> type) {

            return (HibernateProxy.class.isAssignableFrom(

type.getRawType()) ? (TypeAdapter<T>) newHibernateProxyTypeAdapter(gson) : null);

        }

    };

    private final Gson context;

 

    private HibernateProxyTypeAdapter(Gson context) {

        this.context = context;

    }

 

    @Override

    public HibernateProxy read(JsonReader in) throws IOException {

        thrownew UnsupportedOperationException("Not supported");

    }

 

    @SuppressWarnings({"rawtypes", "unchecked"})

    @Override

    public void write(JsonWriter out, HibernateProxy value) throws IOException {

        if(value == null) {

           out.nullValue();

            return;

        }

        // Retrieve the original (not proxy) class

       Class<?> baseType = Hibernate.getClass(value);

        // Get the TypeAdapter of the original class, to delegate the serialization

       TypeAdapter delegate = context.getAdapter(TypeToken.get(baseType));

        // Get a filled instance of the original class

       Object unproxiedValue = ((HibernateProxy) value).getHibernateLazyInitializer()

               .getImplementation();

        // Serialize the value

       delegate.write(out, unproxiedValue);

    }

}

 

然后使用：

Gson gson = newGsonBuilder().registerTypeAdapterFactory(FACTORY).create();

来创建Gson对象.

另外：如果Hibernate中的实体类出现了双向关联，需要转换时排除其中一方，否则会出现由于循环引用导致StackOverflow错误，使用Gson可以通过：

Gson gson = new GsonBuilder()

.registerTypeAdapterFactory(FACTORY)

.setExclusionStrategies(newExclusionStrategy() {               

                     public boolean shouldSkipField(FieldAttributes attr) {

                            // 返回true表示忽略此field

                            return false;

                     }                   

                     public boolean shouldSkipClass(Class<?> clazz) {

                            // 返回true表示忽略此class

                            return false;

                     }

       }).create();

来解决。
