第4讲 条件查询 (WHERE) 语句【重点】

语法：select 列名1，列2 from 表名 where 过滤条件 order by 列 asc|desc

1. 等值 =

   eg：查询工资是17000的员工工号、姓、收入

          select employee_id, last_name, salary from employees where salary=17000;

          查询姓king的员工

          select employee_id, last_name, salary from employees where last_name = 'King';

   注意：在字符串比较时，''中的内容，严格区分大小写。

2.  >  >=  < <=  !=  and  or

       eg：查工资超过10000的员工

              select employee_id, last_name, salary from employees where salary > 10000;

              查工资在10000-20000之间的员工;

              select employee_id, last_name, salary from employees where salary>10000 and  

       salary<20000;

3.  null值的处理：列名 is [not] null 【特别注意】

       eg：查询没有奖金的员工id，姓，工资，提成

       select employee_id, last_name, salary,commission_pct from employees where

       commission_pct is null;  -----> ok

       select employee_id, last_name, salary,commission_pct from employees where

       commission_pct = null;  -----> 结果提示找不到任何数据。

注意：NULL值参与运算总返回NULL，例如：求员工的年薪((sal+comm) * 12)；

      Oracle中NULL 跟''（单引号空串）等价，跟0不等价。

4. 范围查询：列名 [not] between 小值 and 大值

       eg：查工资界于10000(含)-20000(含)之间的员工id，姓和工资

       select employee_id, last_name, salary from employees

       where salary between 10000 and 20000;

              注意：小值在前，大值在后；包含边界值。

5. 枚举查询：列名 [not] in (值1,值2,....值N)

       eg：查在部门60、70、90的员工

              方案一：select employee_id, last_name, salary from employees

              where department_id=60 or department_id=70 or department_id=90;

              方案二：select employee_id, last_name, salary from employees

              where department_id in (60,70,90);

       注意：in的效率较低，开发时不建议使用。

6. 模糊查询：列名 [not] like '格式字符串'

  1）格式字符串：包括字符串常量、通配符(%代表0-N个字符 ;  _代表有且只有1个字符)

   eg：查姓以K开头的员工

              select employee_id, last_name, salary from employees where last_name like 'K%';

           查工资的第二位数是3的员工

              select employee_id, last_name, salary from employees where salary like '__3%';

              查姓的长度是8第3个字母是k的员工

        select employee_id, last_name, salary from employees  where last_name like '__k_____';

              查询98年入的员工    

      select employee_id,last_name,salary from employees where hire_date like '%98';

  2）关于模糊查询中的转义字符

     语法： 列名 like '格式字符串' escape '转义字符' 。在查询时，如果碰到查询条件里含

           有特殊字符时，可以使用转义字符解决问题。但是oracle中没有固定的转义字

           符，如果需要使用，需通过escape关键字声明。

     eg：查询名是以'S_'开头的员工信息。

         select employee_id,first_name,salary from employees

         where first_name like 'S\_%' escape '\';

   注意：

ü  like后的内容，必须放在''中； like是模糊查询,'' 中必须有通配符

ü  模糊查询时，Oracle会将列值转换为字符串后进行比较

第5讲 case ... when 语法结构

基本语法：case...when语句类似于java中的 switch...case 语句，用于根据不同条件返回不同的值，其语法结构为：

   case when exp1 then 结果1 

       when exp2 then 结果2  ....

        else 其他结果

    end

    其中exp1,exp2 , exp3 , exp4 均为布尔表达式。执行流程是如果exp1为真，则返回value1 , 同时忽略对后面的exp2,exp3 的判断。如果exp1为假，则接下来判断exp2。若exp2为真，则返回value2的值，依次类推。如果exp1,exp2,exp3均为假，则返回else后面的value4 。需要注意的是else 并不是必须出现在 case when 结构中的。如果我们没有指定else，同时case  when 结构中所有的条件均为假时，返回NULL。

了解基本用法之后，我们将它运用到SQL语句中：

  select first_name , salary ,

    case

       when salary < 6000 then 'low'

       when salary between 6000 and 10000 then 'middle'

       else 'high'

     end  as "sal_level "

from   ployees
