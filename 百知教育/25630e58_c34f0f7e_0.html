Session的实现机制

Tomcat会为每个拥有session的浏览器设置一个特别的Cookie，称作会话Cookie。会话Cookie的名字固定叫做“JSESSIONID”, Cookie的值就是该浏览器对应的session ID号，其生命周期为-1。

当调用getSession(true)时，他首先检查用户的会话Cookie是否存在

如果不存在：为其创建新的session对象，将session对象的ID保存到新建的会话Cookie中，并将会话cookie送回浏览器 。

如果存在：则按照会话Cookie 的值找到对应的session对象返回

 

Cookie被禁用后实现会话跟踪

在Chrome中禁止使用Cookie：

如果client的Cookie被禁用，则每次请求server，都会重新创建新的session对象。

解决方案：

       String oldPath = "/servlet_day4/session";

       String newPath = response.encodeURL(oldPath);

       encodeURL方法作用：对原URL进行编码，检查client的cookie是否被禁用，如果被禁则自动在原URL后，添加“;jsessionid=xxxxxx”，实现会话。

 

 

Serlvet中的三个作用域对象

知识点

HttpServletRequest

       1）client的请求

              request.setCharacterEncoding("utf-8");

              str = request.getParameter("元素名");

       2）作用域对象

              开始：client写完URL，回车；

              结束：server处理完该请求，响应结果回到client。

              使用：

                     request.setAttribute("name",obj);

                     obj = request.getAttribute("name");

                     request.removeAttribute("name");

       3）使用场景

              逻辑Servlet 给 显示Servlet 传数据，使用forward跳转。

HttpSession

       1）会话，client和server建立连接后的多次请求过程。

              getId()/isNew()/getCreationTime()

       2）作用域对象

              开始：client第1次请求，并且遇到 getSession(true)

              结束：超时/invalidate

              使用：

                     session.setAttribute("name",obj);

                     obj = session.getAttribute("name");

                     session.removeAttribute("name");

       3）使用场景

              同一 client 的多次请求，共享数据时。

ServletContext

       1）是web.xml在内存中的表现。和web应用一一对应。

              ServletContext ctx = super.getServletContext();

              str = ctx.getInitParameter("name");

       2）作用域对象

              开始：server 启动

              结束：server 停止

              使用：

                     ctx.setAttribute("name",obj);

                     obj = ctx.getAttribute("name");

                     ctx.removeAttribute("name");

       3）应用场景

              不同的 client ，共享数据时。

注意：该作用域是servlet中最大的作用域范围，使用时要谨慎。
