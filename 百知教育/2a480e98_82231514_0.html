include包含

场景：多个复杂页面中，存在相同的部分，可以把相同的部分提取出来做成公共页面，然后使用include包含，实现页面的复用。

使用：

              // 获取 请求分发/转发 对象

              RequestDispatcher rd = request.getRequestDispatcher("/要包含的servlet的url-pattern");

              // include包含

              rd.include(request, response);

步骤：

       1) 制作通用页面

              Servlet中的图片显示：<img src='/web应用名/路径/文件名'/>

       2) 在复杂页面中，使用include包含

              RequestDispatcher rd = request.getRequestDispatcher("/要包含的servlet的url-pattern");

              rd.include(request, response);

 

forward 请求转发

场景：显示（界面） 和 （功能）逻辑 分离。

使用：

              // 获取 请求分发/转发 对象

              RequestDispatcher rd = request.getRequestDispatcher("/要包含的servlet的url-pattern");

              // forward请求转发

              rd.forward(request, response);

工作原理图：

forward 请求转发的特点：

              1)服务器内部的转向

              2)地址栏不变

              3)共享同一个request对象

              4)永久转向

              5)只能转向站内资源（同一web应用中）

思考：查询所有帐户时，存在的问题（逻辑Servlet中的局部变量，到显示Servlet中还想使用）

解决方案：在 逻辑Servlet 的 forward 之前，添加：

                     request.setAttribute("str", obj);//往request中设置命名属性

              在 显示Servlet中，使用：

                     Object obj = request.getAttribute("str");//从request中获取命名属性

 

小结，使用forward的3个步骤：

              1) 创建 逻辑Servlet，用于实现功能逻辑处理，通常以Action结尾，如LoginAction

                     // 1、获取client的数据

                     // 2、调用业务类的方法，实现XXX功能

                     // 3、根据返回值，跳转到不同界面

              2) 创建 显示Servlet，用于界面的输出，通常以View结尾，如LoginOkView

                     // 1 设置响应类型

                     // 2 获取输出流

                     // 3使用输出流

              3) 在 逻辑Servlet中，使用 forward 请求转发到 显示Servlet

                     RequestDispatcher rd = request.getRequestDispatcher("/url-pattern");

                     rd.forward(request, response);
