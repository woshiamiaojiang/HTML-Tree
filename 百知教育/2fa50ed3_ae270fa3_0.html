4.  另存文件的方法

Action的File属性接收的文件是临时的，会在action方法执行完毕后删除，需要另外保存：

 
 

// 输入流

 

FileInputStream is = new FileInputStream(aaa);

 

// 得到应用在服务器的真实路径

 

ServletContext sc = ServletActionContext.getServletContext();

 

String path = sc.getRealPath("/day5");

 

System.out.println(path);

 

// 输出流

 

FileOutputStream os = new FileOutputStream(path + "\\" + 文件名);

 

// 读写输入输出流

 

FileUtil.readAndWrite(is,   os);

 

 

5.  文件命名的方法

直接使用文件的原始名很容易就造成文件名称冲突，因此要想办法保证文件名不重复，方法有：

a. 使用毫秒值 new Date().getTime(); // GMT 1970-1-1 0:0:0 （不保险，仍有冲突可能）

b. 从数据库序列获取值

c. 使用uuid：

       java.util.UUID.randomUUID().toString()

d. md5值       （效率低）

       用途：确保文件名唯一(内容唯一)

       md5算法: 只要文件的每个字节是一样的，最后生成的md5值也是一样，最后的结果固定为16个字节     

       另一个用途：加密（不可逆加密）

常见问题

1.  上传路径问题

有人分不清图片上传到服务器的磁盘存储路径，与图片回显路径的区别：磁盘存储路径容易的实现方式是在Tomcat下存储，需要想办法获取Tomcat的实际路径；而图片的回显只是一个URL相对路径，只需要在页面上使用<img src="应用程序名/图片目录/图片名"/> 即可，不需要前面加盘符信息。

2.  输入输出流读写

很多人有畏难心理，一讲到文件另存，涉及输入输出流读写就会犯怵。可以提前将读写工具类提供给学员，或者使用void org.apache.commons.io.FileUtils.copyFile(File srcFile, File destFile) throws IOException来实现文件另存功能，它的底层使用了NIO来实现，效率更优。

注意慎用java.io.File.renameTo(File file)方法来实现文件移动，具体原因可以参考：http://xiaoych.iteye.com/blog/149328
